# Contributing to Libertad VZLA

> **Libertad VZLA is a proprietary project.** Contributions are by invitation only. This document outlines our contributor model and how to express interest.

---

## 📌 Contribution Model

This is **not** an open-source project. We do not accept unsolicited pull requests or public issue reports for feature requests. Our codebase handles sensitive data in a high-risk geopolitical context, and every contributor must be vetted before gaining access.

### Why Invitation-Only?

- **Security:** The platform handles sensitive citizen data and editorial sources that require protection against state-level threats
- **Integrity:** Every contributor must understand and commit to our security protocols, data classification policies, and source protection requirements
- **Quality:** We maintain strict engineering standards (TypeScript strict mode, zero lint errors, comprehensive test coverage)

---

## 🤝 How to Get Involved

If you're interested in contributing, we'd like to hear from you. We're looking for:

### Engineering
- **Full-stack engineers** with Next.js (App Router, RSC, Server Actions) and TypeScript experience
- **Security engineers** for ongoing hardening, penetration testing, and incident response
- **DevOps / Infrastructure** specialists for deployment optimization and monitoring

### Editorial & Content
- **Journalists** with experience covering Venezuela and Latin America
- **Translators** for EN/ES/PT content localization
- **Editors** for editorial workflow management

### Other
- **Security auditors** for independent code and infrastructure reviews
- **Legal advisors** for press freedom and data protection compliance

## 🔐 Contributor Requirements

All accepted contributors must:

1. **Acknowledge the [Security Policy](SECURITY.md)** and commit to its protocols
2. **Follow the [Code Standards](README.md#-code-standards)** — zero exceptions
3. **Use Conventional Commits:** `<type>(<scope>): <description>`
4. **Never commit** secrets, credentials, `.env` files, or PII
5. **Undergo code review** for all changes before merge
6. **Report security concerns** directly to [security@luissambrano.dev](mailto:security@luissambrano.dev) — never in public issues

### Commit Convention

```
type(scope): description

Types:
- feat: New feature
- fix: Bug fix
- refactor: Code refactoring
- style: Code style changes
- docs: Documentation updates
- test: Test additions/modifications
- chore: Maintenance tasks
- security: Security improvements
```

Example: `feat(cms): add role-based article approval workflow`
